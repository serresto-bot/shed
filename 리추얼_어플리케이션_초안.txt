[ Shed 리추얼 어플리케이션 - 기획 초안 ]
※ 기존 브랜드 자료(사쉐 노방 스크립트, shed.html 퀴즈 로직, 리추얼 워크북 레퍼런스 이미지)를 기반으로 정리


━━━━━━━━━━━━━━━━━━━━━━
1. 앱의 역할 (한 줄 요약)
━━━━━━━━━━━━━━━━━━━━━━

사쉐(향)를 받은 사람이,
"향에 맞는 리추얼 루틴을 안내받고 → 2주간 매일 실천 → 스탬프로 인증 → 다 채우면 사쉐 재증정"
을 앱 하나로 관리할 수 있게 하는 것.

= 향은 트리거, 앱은 그 향으로 시작되는 습관을 끝까지 완주시키는 도구.


━━━━━━━━━━━━━━━━━━━━━━
2. 핵심 화면 구성 (총 4개 + 부가 1개)
━━━━━━━━━━━━━━━━━━━━━━

① 홈 (오늘의 리추얼)
② 리추얼 가이드 목록 (프로그램 라이브러리)
③ 리추얼 상세 가이드 (실제 루틴 실행 화면)
④ 스탬프 보드 (2주 완주 현황 + 리워드)
⑤ (부가) 마이페이지 — 내 사쉐 이력 / 완료한 리추얼 기록


━━━━━━━━━━━━━━━━━━━━━━
① 홈 — "오늘의 리추얼"
━━━━━━━━━━━━━━━━━━━━━━

목적: 앱을 켰을 때 "오늘 뭘 해야 하는지" 1초 안에 보이게 함

구성 요소:
- 상단: 날짜, 인사 문구 (예: "오늘도 향 하나, 잠깐 쉬어가요")
- 오늘의 리추얼 카드
  · 현재 진행 중인 향/프로그램명 표시 (예: "Green Breeze 루틴 · 3일차")
  · 오늘 할 3단계 체크리스트 (아래 3번 항목 참고)
  · "오늘 완료하고 스탬프 찍기" 버튼
- 진행률 미니 요약 (점 14개로 표시, 채워진 만큼 진행 상황 한눈에)

※ 체크리스트를 다 채워야 버튼이 활성화되는 구조 제안
  (실천 없이 스탬프만 찍는 걸 방지)


━━━━━━━━━━━━━━━━━━━━━━
② 리추얼 가이드 목록 (프로그램 라이브러리)
━━━━━━━━━━━━━━━━━━━━━━

목적: 내가 받은 향 외에도 다른 리추얼을 둘러보고, 다음 사쉐를 고를 때 참고하게 함

기존 브랜드 향 3종을 그대로 리추얼 프로그램명으로 사용:

┌─────────────────────────────────────────────┐
│ 1) Green Breeze — "환기 리추얼" (아침용)       │
│    한 줄 소개: 아침, 공간의 공기를 바꾸는 루틴   │
│    ① 창문 열고 바람 3번 들이마시기              │
│    ② 따뜻한 음료 한 잔 챙기기                   │
│    ③ 사쉐 향 맡으며 어깨 내려놓기               │
├─────────────────────────────────────────────┤
│ 2) Orange Blossom — "리프레시 리추얼" (낮용)   │
│    한 줄 소개: 낮, 기분을 환기시키는 루틴        │
│    ① 창문 열고 햇빛 2분 받기                    │
│    ② 차갑고 상큼한 음료 한 잔                   │
│    ③ 사쉐 향으로 공간 리프레시                  │
├─────────────────────────────────────────────┤
│ 3) Woody Floral — "나이트 리추얼" (밤용)       │
│    한 줄 소개: 밤, 혼자 조용히 내려놓는 루틴      │
│    ① 폰 뒤집어두기 10분                         │
│    ② 좋아하는 자세로 그냥 있기                  │
│    ③ 사쉐 향 맡으며 오늘 내려놓기               │
└─────────────────────────────────────────────┘

- 내가 받은(구매한) 사쉐 향과 일치하는 프로그램에는
  "내 사쉐와 매칭됨" 배지 표시
- 각 카드 탭하면 ③ 상세 가이드로 이동


━━━━━━━━━━━━━━━━━━━━━━
③ 리추얼 상세 가이드 (실행 화면)
━━━━━━━━━━━━━━━━━━━━━━

목적: 실제로 오늘 그 루틴을 "하게" 만드는 화면 (가이드+실행+기록을 한 화면에)

구성 요소:
- 상단: 향 이름 + 시간대 태그 (예: "AM · 환기 리추얼")
- 제목/한 줄 설명 (감성적 카피, 예: "숲속 이른 아침처럼, 하루를 환기해요")
- 3단계 체크리스트 (번호 매겨서, 순서대로 하나씩 체크)
- 오늘 기분 한 줄 기록 입력창 ("오늘은 왠지...")
  → 폴더의 리추얼 워크북 레퍼런스 이미지(저널형 다이어리)에서 착안.
    "에너지/수면/움직임"까지 다 넣으면 앱이 무거워지므로,
    초안에서는 "한 줄 감정 기록" 하나만 남기고 나머지는 추후 확장 후보로 둠
- "오늘 완료하기 → 스탬프 찍기" 버튼 (누르면 ④ 스탬프 보드에 반영)


━━━━━━━━━━━━━━━━━━━━━━
④ 스탬프 보드 (2주 완주 현황)
━━━━━━━━━━━━━━━━━━━━━━

목적: "내가 얼마나 했는지"를 게임처럼 보여주고, 완주 동기를 주는 화면

구성 요소:
- 상단: 현재 진행 중인 향/루틴명 + 진행 일수 (예: "Green Breeze 루틴 · 14일 중 9일째")
- 14칸 스탬프 그리드 (7일 × 2주)
  · 완료한 날 = 채워진 스탬프 (잎사귀 아이콘 등, 브랜드 톤 유지)
  · 오늘 = 강조 테두리
  · 아직 안 온 날 = 빈 칸/숫자만
- 진행률 바 + "9 / 14 완료" 텍스트
- 완주 리워드 안내 카드
  · "14일 모두 채우면 → 사쉐 1개 무료 재증정"
  · "완료할 때마다 인스타에 인증샷 올리고 #쉐드리추얼 태그해주세요"
    (※ 이 부분이 브랜드 스크립트에서 언급된 "루틴 인증 = 인스타 태그"
       비즈니스 모델과 직접 연결되는 지점 — 반드시 유지)
- 14/14 완료 시 별도 완료 화면
  · "🎉 14일 완성! 새 사쉐가 준비돼요"
  · "재증정 신청하기" 버튼 → (배송지 확인 후 자동 발송 신청 플로우로 연결)


━━━━━━━━━━━━━━━━━━━━━━
⑤ (부가) 마이페이지
━━━━━━━━━━━━━━━━━━━━━━

- 지금까지 받은 사쉐 향 이력
- 완료한 리추얼 프로그램 배지 모음 (Green Breeze 완주, Orange Blossom 완주 등)
- 인스타 인증 히스토리 (링크만 보여주는 정도로 단순하게)
- 디렉터 상담 예약/문의 버튼 (실물 구매 연결 지점)


━━━━━━━━━━━━━━━━━━━━━━
5. 데이터 흐름 (핵심 로직 메모)
━━━━━━━━━━━━━━━━━━━━━━

1) 노방/체험단/구매를 통해 사용자가 특정 향(사쉐)을 받음
2) 디렉터 상담 결과 or 향 선택에 따라, 앱에 "리추얼 프로그램"이 자동 배정됨
3) 사용자는 매일 ③ 상세 가이드에서 체크리스트 완료 → 하루 스탬프 적립
4) 스탬프는 14일(2주) 기준으로 누적
5) 14/14 도달 시 → 인스타 인증 여부 확인 → 사쉐 재증정 신청 오픈
6) 재증정 받으면 새로운 향의 리추얼 프로그램으로 갱신 (루프 반복)

→ 이 로그(어떤 향에 어떤 리추얼을 연결했는지, 완주율이 어떤지)가
   나중에 "AI 자동 리추얼 추천"의 학습 데이터가 됨
   (제품 구조 다이어그램에서 정리했던 "다음 단계" 부분과 연결)


━━━━━━━━━━━━━━━━━━━━━━
6. 아직 결정 안 된 것 (다음 논의 필요)
━━━━━━━━━━━━━━━━━━━━━━

- 인스타 인증을 앱이 자동으로 확인할지, 수동(스크린샷 업로드/사람이 확인)으로 할지
- 하루를 놓치면 스탬프가 끊기는지(연속 실천), 14일 중 누적으로만 세는지(유연하게)
- 디렉터 상담 없이 앱만 다운받은 사람도 리추얼 프로그램을 볼 수 있게 할지
  (체험단 확장 시 무료 리추얼 콘텐츠로 활용 가능)
- 알림(리마인더) 필요 여부 — 매일 정해진 시간에 "오늘의 리추얼" 푸시할지
- 3개 향 프로그램 외 추가 리추얼 프로그램 확장 계획 (계절/이벤트 한정판 등)
